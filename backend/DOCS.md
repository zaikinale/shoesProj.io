# E-Commerce API Documentation

Это RESTful API для платформы электронной коммерции, построенное на **Node.js**, **Express** и **Prisma ORM**. Архитектура проекта следует паттерну **Service-Controller-Router** для обеспечения масштабируемости и чистоты кода.

## Архитектура системы

В проекте используется трехуровневое разделение логики:

1.  **Routes**: Определяют пути (URL) и применяют Middleware для проверки прав доступа.
2.  **Controllers**: Обрабатывают входящие HTTP-запросы, валидируют параметры и формируют ответы.
3.  **Services**: Содержат чистую бизнес-логику и осуществляют запросы к базе данных через Prisma.


## Система аутентификации и ролей (RBAC)

Аутентификация реализована через **JWT (JSON Web Tokens)**, передаваемые в Cookies (`accessToken`).


### Роли пользователей:
| ID | Роль | Описание |
| :--- | :--- | :--- |
| **1** | **User** | Обычный покупатель (Корзина, Заказы, Избранное, Отзывы, Тикеты). |
| **2** | **Manager** | Персонал (Управление товарами, управление статусами заказов). |
| **3** | **Admin** | Полный доступ (Управление ролями, сотрудниками и категориями). |


### Middlewares доступа:
* `authenticateToken`: Парсинг токена. Не прерывает запрос, если токена нет (нужно для публичных витрин).
* `requireAuth`: Прерывает запрос (401), если пользователь не авторизован.
* `authorizeRoles(...ids)`: Прерывает запрос (403), если роль пользователя не совпадает с разрешенными.


## Основные эндпоинты API

### 1. Авторизация (`/api/auth`)
* `POST /register` — Регистрация нового пользователя.
* `POST /login` — Вход (установка cookies).
* `GET /me` — Получение данных о текущем сеансе (требует `requireAuth`).
* `POST /logout` — Выход из системы.

### 2. Товары и Категории (`/api/goods`, `/api/categories`)
* `GET /` — Просмотр списка (публично).
* `POST /` — Создание (только Менеджер/Админ).
* `PUT /:id` — Редактирование (только Менеджер/Админ).
* `DELETE /:id` — Мягкое удаление (только Менеджер/Админ).

### 3. Корзина и Избранное (`/api/basket`, `/api/saves`)
* `GET /` — Получить содержимое (только владелец).
* `POST /add-good` — Добавить товар.
* `DELETE /delete-good/:itemId` — Удалить товар.
* `GET /check/:goodId` — Проверить статус сохранения/наличия.

### 4. Заказы (`/api/orders`)
* `POST /` — Создание заказа из корзины (Транзакция Prisma).
* `GET /my` — История заказов клиента.
* `PUT /:id/status` — Обновление статуса (только Менеджер/Админ).
* `DELETE /:id/cancel` — Отмена заказа клиентом (доступно до отправки).

### 5. Поддержка (Тикеты) (`/api/tickets`)
* `POST /` — Открыть новое обращение.
* `GET /` — Список тикетов (пользователь видит свои, админ — все).
* `POST /:id/messages` — Ответ в тикете.
* `PATCH /:id/close` — Закрытие вопроса.


## Технологический стек
* **Runtime**: Node.js (TypeScript)
* **Framework**: Express.js
* **Database**: PostgreSQL / MySQL (через Prisma)
* **Auth**: JWT, Cookie-parser, Bcrypt
* **Tools**: Prisma Studio (для визуализации БД)


## Установка и запуск

1. Установите зависимости:
   ```bash
   npm install
   ```
2. Настройте переменные окружения в `.env`:
   ```env
   DATABASE_URL="postgresql://user:pass@localhost:5432/bestgames"
    JWT_ACCESS_SECRET="your-secret"
    JWT_REFRESH_SECRET="your-secret"
    PORT=3000
    NODE_ENV=development
   ```
3. Примените миграции базы данных:
   ```bash
   npx prisma migrate dev
   ```
4. Запустите сервер:
   ```bash
   npm run dev
   ```
